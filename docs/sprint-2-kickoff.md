# Sprint 2 Kickoff — AI Product Factory MVP

Date: 2026-06-29
Status: Started

## Sprint 2 goal

Build the first working AI Product Factory MVP that can transform one text requirement into a complete launch pack for a product/store.

## Sprint 2 outcome

By the end of Sprint 2, Genesis AI should be able to produce a structured launch pack containing:

1. Product hypothesis
2. Target customer segment
3. Product catalog
4. Store copy
5. Instagram/Facebook content plan
6. Manual fulfilment checklist
7. 7-day validation plan
8. Phase 2 automation hooks

## Sprint 2 engineering assets to create

### 1. Launch pack schema

Create a reusable JSON/Markdown schema for output generated by the Product Factory agent.

Suggested path:

```text
schemas/launch-pack.schema.json
```

### 2. Example launch pack

Create one realistic example using a selected business idea.

Suggested path:

```text
examples/launch-pack-example.md
```

### 3. Manual runner prompt

Create a single copy-paste prompt that can be used in ChatGPT without API credits.

Suggested path:

```text
prompts/manual-product-factory-runner.md
```

### 4. Execution checklist

Create the step-by-step manual operating checklist for launching the output.

Suggested path:

```text
ops/manual-launch-checklist.md
```

## Sprint 2 constraints

Sprint 2 must respect the lean Phase 1 boundary:

- no required Meta API
- no required Supabase
- no required VPS
- no required Cloudflare
- no required paid API credits

## Sprint 2 first task

Create the launch pack schema, then create the manual runner prompt.

## Done criteria

Sprint 2 is done when a plain-text input can produce a repeatable launch pack and the founder can execute it manually without additional infrastructure.
